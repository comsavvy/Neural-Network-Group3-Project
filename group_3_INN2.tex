\section{Introduction}
Object detection is a fundamental task in computer vision that involves identifying and localizing objects within an image. In this project, we implemented the YOLOv5 algorithm to perform object detection on the Pascal VOC 2012 dataset, which contains images of 20 different object classes. The goal was to train a model that can accurately detect and classify objects in the images, and to evaluate its performance using standard metrics.


%% ==================  DATASET  ============================
\section{Dataset}

\subsection{Pascal VOC 2012}

The Pascal VOC 2012 dataset is a standard benchmark for object detection, containing real-world images annotated across 20 object classes such as person, car, dog, and bicycle.

For this project, the dataset is used in the YOLO format, where each image is associated with a text file containing normalized bounding box annotations. This format is directly compatible with PyTorch-based YOLO implementations, requiring no additional conversion.

Since the data was stored locally, a custom helper module \texttt{YOLODataset} was used to load images and corresponding labels, ensuring proper parsing of annotations and integration with the training pipeline.

It is important to note that the dataset exhibits class imbalance, with certain classes (such as \textit{person}) appearing significantly more frequently than others. %===========================================================

\section{Overview of You Only Look Once (YOLO)}

You Only Look Once (YOLO) is a \textbf{real-time object detection} algorithm that formulates detection as a single regression problem. Instead of applying a classifier at multiple locations, YOLO processes the entire image in one forward pass of a neural network.

\subsection{Definitions}

\textbf{Object Detection:} The task of identifying and localizing objects in an image. Localization is typically done using \textbf{bounding boxes}, which are rectangles defined by their center coordinates, width, and height.

\textbf{Grid:} The input image is divided into an $S \times S$ grid. Each grid cell is responsible for detecting objects whose center lies within it.

\textbf{Bounding Box:} A rectangle described by $(x, y, w, h)$, where $(x,y)$ is the center, and $(w,h)$ are width and height, usually normalized by the image size.

\textbf{Confidence Score:} Measures how likely a bounding box contains an object and how accurate the box is. It is often defined as:
\[
\text{Confidence} = P(\text{object}) \times \text{IoU}
\]

\textbf{Intersection over Union (IoU):} A metric that measures the overlap between the predicted box and the ground truth:
\[
\text{IoU} = \frac{\text{Area of Overlap}}{\text{Area of Union}}
\]

\textbf{Class Probability:} Each grid cell predicts probabilities for each object class, representing the likelihood that a detected object belongs to a given category (output)

\subsection{YOLO Pipeline}

\begin{itemize}
	\item The image is divided into an $S \times S$ grid.
	\item Each grid cell predicts:
	\begin{itemize}
		\item $B$ bounding boxes,
		\item Confidence scores for each box,
		\item Class probabilities.
	\end{itemize}
	\item Predictions are combined to produce final detections.
	\item \textbf{Non-Maximum Suppression (NMS)} is applied to remove duplicate boxes by keeping only the highest-confidence predictions.
\end{itemize}


\section{Model Training and Evaluation}
\subsection{YOLO Loss Function}

The implemented loss function follows a grid-based formulation and is computed over all anchors. Let $\hat{y}$ denote predictions and $y$ the ground truth. For each anchor $b \in \{1,\dots,B\}$, the prediction vector is decomposed into:
\[
(\hat{x}, \hat{y}, \hat{w}, \hat{h}, \hat{c}, \hat{p}_1, \dots, \hat{p}_C)
\]

An indicator variable $obj \in \{0,1\}$ specifies whether an object is present in a given grid cell.

\begin{itemize}
	\item \textbf{Localization Loss:} Applied only when an object is present ($obj = 1$), with sigmoid activation on coordinates and square root transformation on width and height:
	\[
	\mathcal{L}_{xy} = \lambda_{\text{coord}} \sum obj \left[ (\sigma(\hat{x}) - x)^2 + (\sigma(\hat{y}) - y)^2 \right]
	\]
	\[
	\mathcal{L}_{wh} = \lambda_{\text{coord}} \sum obj \left[ (\sqrt{\hat{w}} - \sqrt{w})^2 + (\sqrt{\hat{h}} - \sqrt{h})^2 \right]
	\]
	
	\item \textbf{Confidence Loss:}
	\[
	\mathcal{L}_{\text{obj}} = \sum obj \, (\sigma(\hat{c}) - c)^2
	\]
	\[
	\mathcal{L}_{\text{noobj}} = \lambda_{\text{noobj}} \sum (1 - obj)\, \sigma(\hat{c})^2
	\]
	
	\item \textbf{Classification Loss:} Computed only for cells containing objects:
	\[
	\mathcal{L}_{\text{class}} = \sum obj \sum_{k=1}^{C} (\hat{p}_k - p_k)^2
	\]
\end{itemize}

The total loss is obtained by summing over all anchors and averaging over the batch:
\[
\mathcal{L} = \frac{1}{N} \sum_{b=1}^{B} \left(
\mathcal{L}_{xy} + \mathcal{L}_{wh} + \mathcal{L}_{\text{obj}} + \mathcal{L}_{\text{noobj}} + \mathcal{L}_{\text{class}}
\right)
\]

where $\lambda_{\text{coord}}$ and $\lambda_{\text{noobj}}$ are weighting parameters controlling the contribution of localization and no-object confidence terms.
\subsection{Training Procedure}

For computational efficiency, the model is trained on a subset of the dataset, with all images resized to a fixed resolution. The network is optimized using the Adam optimizer with a learning rate of $10^{-3}$ over a fixed number of epochs.

At each iteration, the ground truth annotations are transformed into the YOLO grid-based representation using a helper function \texttt{encode\_target}, which maps bounding boxes and class labels into a tensor of shape $S \times S \times (B \cdot 5 + B \cdot C)$.

Here, $S$ denotes the grid size (the image is divided into an $S \times S$ grid), $B$ is the number of bounding boxes predicted per grid cell, and $C$ is the number of object classes. The term $5$ corresponds to the parameters $(x, y, w, h, c)$, where $(x, y)$ are the box center coordinates, $(w, h)$ its width and height, and $c$ the confidence score.

The model predictions are then compared to the encoded targets using the YOLO loss function, and parameters are updated via backpropagation.


\subsection{Evaluation}

The model is evaluated by computing the mean YOLO loss over different datasets. In evaluation mode, no gradient updates are performed, and the loss is averaged across all batches.

The mean loss values obtained are:
\begin{itemize}
	\item Training subset (500 images) $\approx$  12.4
	\item Validation set $\approx$ 12.7
	\item Augmented training subset $\approx$ 11.2
	\item Augmented Validation set $\approx$ 13.2
\end{itemize}

\section{Advantages and Limitations}

\subsection{Advantages of YOLO}
\begin{itemize}
	\item \textbf{Real-time detection:} YOLO performs object detection in a single forward pass, making it significantly faster than region-based methods.
	
	\item \textbf{Global reasoning:} The model processes the entire image at once, allowing it to capture contextual information and reduce false positives.
	
	\item \textbf{End-to-end training:} Detection is formulated as a single regression problem, simplifying the pipeline compared to multi-stage approaches.
\end{itemize}

\subsection{Limitations of YOLO}
\begin{itemize}
	\item \textbf{Localization errors:} The grid-based structure can lead to imprecise bounding boxes, especially for small objects.
	
	\item \textbf{Limited capacity per grid cell:} Each cell predicts a fixed number of boxes, which may cause missed detections when multiple objects are close together.
	
	\item \textbf{Sensitivity to small objects:} Objects occupying a small portion of the image are harder to detect due to spatial downsampling.
\end{itemize}

\subsection{Project-specific considerations}
\begin{itemize}
	\item \textbf{Limited dataset size:} Training on a small subset restricts the model's ability to generalize and may lead to overfitting.
	
	\item \textbf{Reduced image resolution:} Images were resized for faster computation, which can degrade detection performance, especially for fine details.
	
	\item \textbf{Computational constraints:} The model was trained on limited hardware, requiring simplifications in architecture and training duration, which impacts overall accuracy.
	
	\item \textbf{Class imbalance bias:} The prevalence of the \textit{person} class can bias the model towards detecting people more frequently than other classes.
\end{itemize}
